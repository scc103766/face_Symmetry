# FaceSymAi 人脸对称性判断算法研发计划

版本：0.2  
日期：2026-05-19  
状态：领导汇报版  

## 1. 汇报摘要

本研发计划将 FaceSymAi 当前阶段收敛为 **两个月完成 V1 静态分析版本**：

- 关键点检测：确定使用 **MediaPipe Face Landmarker**。
- 输入范围：正脸静息图、露齿图。
- V1 能力：人脸总体对称性评分，口部、眼部、眉部、鼻面中线、面部轮廓等部件级对称性属性分析。
- 输出形式：可被外部系统调用的 API，返回质量门控、部件特征、总体评分、疑似异常侧、解释项和卒中预警辅助信号。
- 验收口径：冻结测试集上输出准确、可复核的 `precision = TP / (TP + FP)`。
- V2 方向：视频动态运动特征、动作时序特征和深度融合模型，不纳入两个月 V1 主交付。

## 2. V1 研发目标

V1 的目标不是完成复杂临床诊断模型，而是在两个月内交付一个稳定、可解释、可评估的静态图片算法 API。

### 2.1 输入

- 正脸图：静息状态，无遮挡，光照均匀。
- 露齿图：正脸露齿或咧嘴，口角和口型清晰。

### 2.2 输出

- `overall_symmetry_score`：人脸总体对称性评分。
- `affected_side`：疑似异常侧。
- `quality_score`：输入质量分。
- `mouth_asymmetry`：口部对称性属性。
- `eye_asymmetry`：眼部对称性属性。
- `brow_asymmetry`：眉部对称性属性。
- `midline_asymmetry`：鼻面中线对称性属性。
- `contour_asymmetry`：面部轮廓对称性属性。
- `stroke_warning_auxiliary_score`：卒中预警辅助信号。
- `top_contributions`：主要异常贡献项。
- `recommended_action` 和 `disclaimer`：复核建议和医学边界声明。

### 2.3 不纳入 V1 的内容

- 视频动态运动特征。
- 动作时序模型。
- 深度融合模型。
- 基于短视频的闭眼、抬眉、鼓腮动态分析。
- 临床诊断结论。

这些内容进入 V2。

## 3. 两个月时间规划

两个月按 8 周执行，目标是在第 8 周完成 V1 API、评估报告和汇报演示。

| 周期 | 阶段 | 目标 | 关键交付 |

| 第 2 周 | MediaPipe 接入 | 完成图片关键点检测和语义关键点映射 | MediaPipe adapter、正脸/露齿关键点 JSON |

## 4. 阶段任务拆解

### 4.1 第 1 周：需求冻结与数据盘点

目标：把两个月内能交付的 V1 范围固定下来。

工作项：

- 确认 V1 只支持正脸图和露齿图。
- 确认 MediaPipe 是唯一 V1 关键点检测方案。
- 确认 V1 输出字段和 API schema。
- 确认 precision 验收口径。
- 盘点现有 xlsx 是否存在标签、审核结果和图片/视频关联字段。
- 设计冻结测试集字段：`sample_id`、`patient_id_hash`、`input_type`、`label`、`quality_label`。

交付物：

- `docs/algorithm/evaluation-protocol.md` 更新确认。
- V1 输入输出字段清单。
- 测试集 manifest 草案。

验收：

- V1/V2 边界清晰。
- precision 计算方式明确。
- 不再把视频动态模型放入 V1 交付。

### 4.2 第 2 周：MediaPipe 接入

目标：把正脸图和露齿图转成 FaceSymAi 标准关键点结构。

工作项：

- 接入 MediaPipe Face Landmarker。
- 封装图片检测接口。
- 输出 bbox、pose、landmarks、blendshape、检测置信度。
- 建立 MediaPipe 关键点到 FaceSymAi 语义点的映射。
- 输出标准 `FaceLandmarks` schema。
- 建立检测失败、多人脸、关键点缺失错误码。

交付物：

- `src/facesymai/detectors/mediapipe_adapter.py`。
- `src/facesymai/landmark_mapping.py`。
- `tests/test_mediapipe_adapter.py`。
- 正脸图和露齿图样例 JSON。

验收：

- 正脸图可输出眼、眉、鼻、口、轮廓关键点。
- 露齿图可输出口角、上下唇、唇中点等口部关键点。
- MediaPipe 不可用时，原有关键点 JSON baseline 仍可运行。

### 4.3 第 3 周：质量门控 V1

目标：保证低质量图片不会进入高置信评分流程。

工作项：

- 单人脸检测。
- 人脸大小检测。
- 姿态 yaw/pitch/roll 门控。
- 模糊检测。
- 光照检测。
- 核心区域遮挡检测。
- 关键点置信度检测。
- 露齿图合规检测。

交付物：

- `src/facesymai/quality.py`。
- `tests/test_quality_gate.py`。
- 质量门控错误码文档。

验收：

- 无人脸、多人脸、侧脸、遮挡、模糊、低光能返回明确原因。
- API 中质量分和算法分分开输出。
- 质量不达标样本返回重采建议。

### 4.4 第 4 周：静态特征 V1-A

目标：完成总体对称性、口部、鼻面中线三类核心特征。

工作项：

- 全局镜像误差。
- 左右口角高度差。
- 左右口角到中线距离差。
- 露齿图口型/口角外展不对称。
- 上下唇中点偏移。
- 鼻尖、唇中点、下巴相对中线偏移。

交付物：

- `src/facesymai/features_static.py`。
- `src/facesymai/features_mouth.py`。
- `src/facesymai/features_midline.py`。
- 单元测试和合成样例。

验收：

- 对称样本输出高总体对称分。
- 口角下垂样本输出口部异常和疑似异常侧。
- 中线偏移样本输出鼻面中线异常解释。

### 4.5 第 5 周：静态特征 V1-B

目标：补齐眼部、眉部、面部轮廓属性，并完成部件级聚合。

工作项：

- 左右眼裂高度差。
- 左右眼宽和眼角高度差。
- 左右眉头、眉尾高度差。
- 左右眉眼距离差。
- 左右脸颊和下颌轮廓镜像误差。
- 部件级 severity 归一化。
- `affected_side` 聚合。
- `top_contributions` 解释排序。

交付物：

- `src/facesymai/features_eye.py`。
- `src/facesymai/features_brow.py`。
- `src/facesymai/features_contour.py`。
- `src/facesymai/side.py`。
- `src/facesymai/scoring.py`。

验收：

- 口部、眼部、眉部、中线、轮廓都能独立输出属性分。
- 总体评分由部件级分数加权得到。
- 解释项能指出主要异常部件和方向。

### 4.6 第 6 周：API 与报告输出

目标：形成外部可调用 API。

工作项：

- 封装 V1 pipeline。
- 支持正脸图、露齿图上传。
- 输出标准 JSON。
- 增加模型版本、特征版本、质量门控版本。
- 增加错误码、重采建议和免责声明。
- 生成 API 文档和样例。

交付物：

- `src/facesymai/api/`。
- `src/facesymai/pipeline.py`。
- `docs/api/facial-symmetry-api.md`。
- `tests/test_api_contract.py`。

验收：

- API 可接收正脸图和露齿图。
- API 可返回完整对称性结果。
- 错误输入可返回稳定错误码。

### 4.7 第 7 周：测试集评估

目标：在冻结测试集上完成 precision 评估。

工作项：

- 固定测试集版本。
- 固定阈值和阳性判定规则。
- 批量运行 API 或离线 pipeline。
- 输出样本级预测明细。
- 计算 precision、TP、FP、confusion matrix。
- 进行误报样本归因。

交付物：

- `reports/v1_precision_report.md`。
- `reports/v1_prediction_details.csv`。
- `reports/v1_error_analysis.md`。

验收：

- precision 可由样本级结果复核。
- 报告包含 TP、FP、预测阳性数、测试集总量、阈值和版本信息。
- 误报原因有分类统计。

### 4.8 第 8 周：验收与汇报

目标：完成 V1 交付和领导汇报。

工作项：

- 准备 API 演示样例。
- 汇总 precision 评估报告。
- 汇总风险与限制。
- 输出 V2 规划和资源需求。

交付物：

- V1 验收报告。
- V1 API 演示。
- V1 precision 报告。
- V2 研发建议。

验收：

## 6. V1 验收标准

### 6.1 功能验收

- 正脸图和露齿图可完成 MediaPipe 关键点检测。
- 可输出总体对称性评分。
- 可输出口部、眼部、眉部、鼻面中线、面部轮廓对称性属性。
- 可输出疑似异常侧和主要解释项。
- 可输出质量门控结果和重采建议。
- 可通过 API 返回稳定 JSON。

### 6.2 指标验收

主指标：

```text
precision = TP / (TP + FP)
```

报告必须包含：

- 测试集版本。
- 模型/API 版本。
- 阈值和阳性规则。
- TP、FP、预测阳性数量。
- precision 数值。
- confusion matrix。
- 样本级预测明细。

### 6.3 工程验收

- 核心流程可复现。
- 单元测试覆盖质量门控和主要特征。
- API 契约测试通过。
- 所有输出包含医学边界声明。
- 低质量输入不会输出高可信预警。

## 8. V2 预留方向

V2 在 V1 验收后启动，重点提升动态识别和模型鲁棒性。

V2 研发内容：

- 短视频输入。
- 露齿微笑动态幅度。
- 闭眼不全检测。
- 抬眉动态对称性。
- 鼓腮动作分析。
- 自然眨眼左右同步性。
- 动作时序特征：幅度、速度、延迟、峰值时间。
- 关键点时序模型：TCN、GRU、Transformer。
- 图像深度模型：CNN/ViT ROI 分支。
- 深度融合模型：静态特征 + 动态特征 + 图像 embedding + 质量信息。
- 概率校准和外部验证。

V2 的目标是把 V1 的静态辅助评分升级为更稳健的动态面瘫识别和卒中预警辅助模型。

## 9. 主要风险与控制

| 风险 | 影响 | 控制措施 |
| --- | --- | --- |
| 两个月周期过短 | 范围失控 | V1 只做静态图，不做视频模型 |
| MediaPipe 在低质图上漂移 | 特征误判 | 强质量门控，低置信区域不评分 |
| 露齿图不标准 | 口部评分不稳定 | 增加露齿合规检测和重采建议 |
| 仅用静态图召回不足 | 漏检风险 | 明确 V1 边界，V2 加动态视频 |
| 标签不足 | precision 无法定义 | 先明确测试集阳性标签，必要时先评估面瘫/对称性 precision |
| 医疗表述风险 | 合规风险 | 全部输出使用“辅助预警”，不使用“诊断” |

## 10. 对领导汇报的核心口径

建议汇报时使用以下口径：

1. 两个月内交付 V1 静态图片分析 API。
2. V1 使用 MediaPipe，专注正脸图和露齿图。
3. V1 重点输出总体对称性评分和五类部件属性：口部、眼部、眉部、鼻面中线、面部轮廓。
4. V1 验收用冻结测试集 precision，指标可复核。
5. V1 不做临床诊断，只做面瘫/卒中预警辅助解释。
6. V2 再做视频动态、动作时序和深度融合模型。
